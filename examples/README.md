# examples 目录说明

`examples/` 保存可提交、可分享的样例数据和样例输出。这里的数据应脱敏，便于别人理解项目格式，而不会泄露真实业务内容。

| 路径 | 功能 |
|---|---|
| `output/` | 示例输出目录，可放脱敏后的 JSON、CSV、Excel 或 HTML |

## 使用建议

- 从 `data/` 或 `output/` 复制样例到这里前，先删除真实公司、客户、价格、人员、群聊和凭证信息。
- 示例文件名尽量体现来源和用途，例如 `sample_tasks_prepared.json`。
- 如果新增一种输入或输出格式，优先在这里放一个最小样例。
